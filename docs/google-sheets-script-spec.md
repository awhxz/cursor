# ТЗ: Google Apps Script для валидации и перемещения задач

## Цель

Скрипт `google-apps-script/task-routing.gs` встраивается в Google Sheets через **Расширения → Apps Script**. Он не нужен Vercel и не входит в код Next.js.

## Правила

1. Номера столбцов не зашиты в логику: скрипт ищет их по заголовкам в первой строке.
2. При вводе в «Тикет» разрешается пустое значение или URL, начинающийся с `http://`/`https://`. Иной ввод отклоняется.
3. В «Ценность» и «Время в аналитике» разрешены `1`, `2`, `3`, `4` или пустая ячейка.
4. При изменении статуса обязательно заполнены: «Тикет», «Название», «Заказчик», «Ответственный», «К чему относится», «Ценность», «Время в аналитике».
5. Если валидация не пройдена, статус возвращается к прежнему значению (или очищается при массовой вставке), строка не перемещается, а пользователь видит toast-сообщение.
6. Валидная строка перемещается между «Актуальные», «Готовые задачи» и «Общий бэклог» по маппингу из ТЗ дашборда.
7. При копировании строки переносятся значения, формулы, форматы и data validation, после чего исходная строка удаляется.
8. Массовая реорганизация пропускает невалидные строки и показывает отдельные счетчики перемещенных и пропущенных задач.

## Установка

1. Скопировать содержимое `google-apps-script/task-routing.gs` в Apps Script.
2. Сохранить проект.
3. Обновить Google Sheets: появится меню «📋 Управление задачами».
4. Один раз запустить «⚙️ Настроить валидацию» и выдать скрипту разрешения.
5. Проверить на копии таблицы: неверную ссылку, пустые обязательные поля, каждый статус и массовую реорганизацию.
